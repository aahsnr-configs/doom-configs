Creating Daily TODOs
in Doom Emacs

A practical guide to capturing tasks, managing states, scheduling work, and reviewing your day — all through Doom's default keybindings. No custom config required to follow along.

Updated June 2026

01 · Setup

Setup & Prerequisites

Doom Emacs ships the :lang org module — Org-mode's integration layer — enabled by default in most fresh installs. Before anything else, confirm that the module is active in your ~/.doom.d/init.el (or ~/.config/doom/init.el):

init.el 
;; Inside the :lang block — ensure org is present
:lang
(org              ; minimum required
 ;; optional flags — add what suits you:
 +pretty          ; fancy bullets & priorities
 +roam2           ; knowledge graph (requires sqlite3)
 +journal         ; org-journal integration
 +pomodoro)       ; built-in timer for tasks

After editing init.el, run doom sync in your terminal (or SPC h r r inside Doom) to install any missing packages and regenerate the autoloads file.

ℹ
Keybinding Notation This guide uses evil (vim-style) keybindings — Doom's default mode. SPC is the leader key. SPC m opens the local leader (major-mode commands). In insert mode, substitute SPC with M-SPC. If you've disabled evil, replace SPC with C-c and SPC m with C-c l.
02 · Files

Org Directory & Files

Doom sets org-directory to ~/org/ by default. Every TODO you capture lands in files within this directory. It is best practice to configure this before org loads — i.e., at the very top of ~/.doom.d/config.el:

config.el 
;; Must be set BEFORE org loads
(setq org-directory "~/org/")

Doom's built-in capture templates create three key files automatically inside your org-directory:

Doom Default Org Files
FilePurpose
todo.org Personal TODO items captured via SPCXt
notes.org Freeform notes captured via SPCXn
journal.org Date-tree journal entries captured via SPCXj

You can open any of these files directly with the Vim-style ex-command :e todo.org or with SPCff and typing the filename — Doom resolves it relative to org-directory automatically.

✦
Add files to the agenda For your TODO items to show up in the Org Agenda, the files must be listed in org-agenda-files. Doom pre-populates this with your org-directory, but you can extend it:

(setq org-agenda-files (list org-directory "~/work/projects.org"))
03 · Capture

org-capture: Fast TODO Capture

org-capture is the fastest way to add a TODO without losing your current context. You invoke it from anywhere in Doom — even in the middle of writing code — and it pops up a temporary buffer where you type your task. When you finish, the item is filed automatically and you return exactly where you were.

Capture Keybindings
KeybindingAction
SPCX
Open the capture dispatcher menu Lists all available templates — pick with a single key press.
SPCX t
Capture a personal Todo → todo.org Creates a * TODO headline with timestamp.
SPCX n
Capture a personal Note → notes.org
SPCX j
Capture a Journal entry → journal.org Organized under a date-tree (Year → Month → Day).
C-c C-c
Confirm & file the capture entry Works in the capture buffer. Also aliased to ZZ in evil mode.
C-c k
Abort the capture without saving
C-c w
Refile the entry to a different location before confirming

What a captured TODO looks like

After pressing SPCXt and typing your task title, Doom writes the following structure to todo.org:

todo.org 
* TODO Review the pull request for feature/auth-refactor
:PROPERTIES:
:ID:       a1b2c3d4-...
:END:
[2026-06-23 Tue 14:35]
✦
Capture from anywhere You can invoke capture while browsing a file in Dired, inside a Magit diff, or mid-code-edit. Doom's capture is non-destructive — it does not move your cursor, does not close your current buffer, and restores your window layout after C-c C-c.
04 · Inline

Writing TODO Headings Directly

Sometimes you'd rather be directly inside an org file building out a structured task list instead of going through the capture dispatcher. Once you have an org buffer open (todo.org, or any *.org file), these are the essential editing keybindings.

Heading & TODO Creation
KeybindingAction
C-RET Insert a new heading at the same level below the current one Most efficient way to add sibling TODO items quickly.
C-S-RET Insert a new heading and immediately toggle it to TODO state Creates a TODO sibling in one step — the classic fast-entry combo.
SPCmt Toggle / cycle the TODO keyword on the current heading On a non-TODO heading: sets it to TODO. On TODO: advances the state.
S-→ Cycle TODO state forward (TODO → NEXT → DONE…)
S-← Cycle TODO state backward
M-S-RET Insert a new TODO heading at the end of the current subtree
C-c C-x b Toggle checkbox (if cursor is on a checkbox item)
SPCmx Toggle org checkbox on the current line Use - [ ] task syntax to create checkbox lists under headings.

Promoting & Demoting Headings

Organize your tasks into projects and sub-tasks by controlling heading levels:

Heading Level Control
KeybindingAction
M-S-→ Demote heading (indent one level deeper — makes it a child)
M-S-← Promote heading (outdent — makes it a parent / sibling)
M-↑ Move the current subtree up past the previous sibling
M-↓ Move the current subtree down past the next sibling
TAB Cycle visibility of the current subtree (fold/unfold)
S-TAB Cycle visibility of the entire buffer globally
✎
Checkboxes vs. TODO headings Use checkboxes (- [ ] step) for sub-steps inside a single task. Use TODO headings (** TODO) for discrete items that may need scheduling, priorities, or their own due dates. Checkboxes live inside a heading's body; they don't appear in the agenda.
05 · States

TODO States & Cycling

Doom's default org-todo-keywords include a rich set of states split across two sequences — active (un-done) and completed. Understanding the flow between states is central to effective daily task management.

Default Doom TODO keyword sequences

TODO → PROJ → STRT → WAIT → HOLD → DONE / KILL

The second sequence handles checkbox-like states for lists embedded inside headings: [ ] → [-] → [X].

State Cycling Keybindings (in an org buffer)
KeybindingAction
SPCmt Open keyword selection menu (cycle or choose from a list) Doom shows a popup picker when you have many keyword states.
S-→ Advance state forward (TODO → PROJ → STRT → …)
S-← Retreat state backward
C-c C-t Vanilla Emacs state-change (works regardless of evil mode) Prompts for a single-letter shortcut from the keyword sequence.

State meanings at a glance

State Reference
StateMeaning
TODO An actionable item not yet started. The standard starting state.
PROJ A container / project heading with child TODO items beneath it.
STRT Work In Progress — this is what you're actively doing right now.
WAIT Blocked on someone else; you can't proceed without external input.
HOLD Deliberately paused — your decision, not an external blocker.
DONE Completed. Org records a CLOSED: timestamp automatically.
KILL Cancelled, dropped, or no longer relevant. Not the same as DONE.
ℹ
Logging state changes By default Doom logs a CLOSED: timestamp when a task moves to DONE. You can enable richer logging (notes on state change, timestamps on every transition) by adding (setq org-log-done 'note) to your config.el.
06 · Scheduling

Scheduling & Deadlines

The distinction between SCHEDULED and DEADLINE is important in org-mode. A scheduled date is when you plan to start working on the task; a deadline is when it must be done. Both appear in the Org Agenda, but with different visual urgency as the date approaches.

Date & Time Keybindings (cursor on a TODO heading)
KeybindingAction
SPCmds Set the SCHEDULED date Opens Emacs calendar picker. Navigate: S-←/→ moves by day, </> moves by month.
SPCmdd Set the DEADLINE date Deadline warnings start appearing in the agenda N days before. Default is 14 days.
SPCmdt Insert an active timestamp <YYYY-MM-DD> Active timestamps appear in the agenda as events (not todos).
SPCmdT Insert an inactive timestamp [YYYY-MM-DD] Inactive timestamps do NOT appear in the agenda — useful for notes.
C-c C-s Vanilla binding: schedule the heading (same as above)
C-c C-d Vanilla binding: set deadline on the heading

Inside the date picker

When the calendar popup opens after pressing a scheduling keybinding, use these keys to navigate to the desired date before pressing RET:

Calendar Picker Navigation
KeyAction
S-← / → Move backward / forward one day
S-↑ / ↓ Move backward / forward one week
< / > Move backward / forward one month
. Jump to today
RET Confirm the selected date
Type +2d, +1w, +3m Relative date input — "2 days from now", "1 week from now", etc.

What it looks like in the file

todo.org — scheduled + deadline example 
* TODO [#A] Ship authentication module :work:backend:
  DEADLINE: <2026-06-30 Tue>  SCHEDULED: <2026-06-24 Wed>
  :PROPERTIES:
  :Effort:   2h
  :END:
  Implement JWT refresh flow, write integration tests,
  coordinate deploy window with DevOps.

Recurring tasks

For tasks that repeat — daily standups, weekly reviews — add a repeater interval to the timestamp directly in the org file:

Repeater Syntax 
* TODO Daily standup
  SCHEDULED: <2026-06-24 Wed 09:00 +1d>   ; repeat every day

* TODO Weekly team retrospective
  SCHEDULED: <2026-06-27 Sat .+1w>    ; schedule 1w from when DONE

* TODO Monthly financial review
  SCHEDULED: <2026-07-01 Wed ++1m>    ; catch-up repeater
ℹ
Repeater types +1d schedules exactly 1 day ahead of the original date each time. .+1w schedules 1 week from the date you marked it DONE (best for habit-style tasks). ++1m advances until the next future occurrence (good for catch-up if you miss cycles).
07 · Priorities & Tags

Priorities & Tags

Priorities let you rank TODO items. Doom uses three levels by default: [#A] (urgent), [#B] (normal, the implicit default), and [#C] (low). Tags help you filter and group tasks in the agenda by context, project, or person.

Priority Keybindings
KeybindingAction
S-↑ Raise priority of the current heading (C → B → A)
S-↓ Lower priority (A → B → C)
C-c , Set priority interactively (prompts A, B, C, or SPC to clear)
SPCmp Set priority via Doom's local leader menu
Tag Keybindings
KeybindingAction
SPCmq Add or edit tags on the current heading Autocompletes from existing tags in your org files.
C-c C-q Vanilla tag insertion / editing
C-c C-c When cursor is on a heading line, opens tag editor Same key as "execute block" — context-sensitive.

Tags are colon-delimited and appear at the right margin of the heading:

Tag syntax examples 
* TODO [#A] Fix login page bug          :work:frontend:urgent:
* TODO Buy groceries                    :personal:errands:
* TODO [#C] Read "Thinking Fast & Slow"  :reading:low_priority:
✦
Context tags for GTD-style filtering Many users adopt GTD "context" tags like :@home:, :@office:, :@phone:. These become powerful filters in the Org Agenda — press m inside the agenda to filter by tag.

Effort estimates

Set how long a task should take using the Effort property. This feeds into the agenda column view and clocktable reports:

Effort Property
KeybindingAction
SPCmo Set a property on the current heading (choose Effort from the picker)
C-c C-x e Set effort estimate directly (vanilla keybind)
08 · Agenda

The Org Agenda

The Org Agenda is your daily command center. It aggregates all scheduled tasks, deadlines, and active TODOs from your org files into a unified calendar-plus-list view. This is where you review what needs doing today and take action on items.

Opening the Agenda
KeybindingAction
SPCoa Open the Org Agenda dispatcher menu Presents a menu of agenda views to choose from.
SPCoa then a Daily / weekly agenda — your primary view Shows today's scheduled items and upcoming deadlines.
SPCoa then t Global TODO list — all TODO-state items across all org files
SPCoa then m Tag / property match — filter by tag expression E.g. type +work-done for work items not yet done.

Navigating inside the Agenda buffer

Once the agenda is open, a rich set of single-key commands becomes available. You do not need a leader prefix — just press the key directly in normal mode:

Agenda Buffer Navigation & Actions
KeyAction
j / k Move cursor down / up through agenda items (evil-style)
n / p Next / previous day (or week, depending on the current view span)
RET Jump to the org file and heading for the item under the cursor
o Open the entry in another window without leaving the agenda
t Cycle TODO state of the item under cursor directly from the agenda Mark something DONE without leaving the agenda view.
s Schedule the item under cursor
d Set / change deadline for item under cursor
+ / - Raise / lower priority of the item under cursor
m Filter by tag Type a tag name (e.g. work) and press RET to narrow the view.
/ Filter by regex (narrows agenda to matching headlines)
| Remove all active filters and show full agenda again
r or g Refresh / rebuild the agenda view
vd Switch to day view (show only today)
vw Switch to week view
vm Switch to month view
q Close / quit the agenda buffer
⚠
Keep org-agenda-files in sync If you create new .org files outside org-directory, they won't appear in the agenda automatically. Add them to the list in your config.el or use M-x org-agenda-file-to-front while visiting the file.
09 · Clocking

Clocking Time on Tasks

Org-mode's built-in time clock lets you track exactly how long you spend on each task. This builds an accurate picture of your daily output and helps you calibrate future effort estimates. The clock writes a LOGBOOK drawer into the heading automatically.

Clock Keybindings (cursor on a TODO heading)
KeybindingAction
SPCmci Clock in — start the timer on this task Displays running clock in the mode line: [1:23]
SPCmco Clock out — stop the current timer Writes the CLOCK interval to the LOGBOOK drawer.
SPCmcx Cancel the running clock (discards the time)
SPCmcr Insert a clocktable report at the cursor Generates a summary table of all clocked time in the current file or subtree.
SPCmcj Jump to the currently clocked-in task (from anywhere in Emacs)
SPCmce Set effort estimate for the current task

What the LOGBOOK looks like

todo.org — clocked task 
* DONE [#B] Implement rate limiting middleware  :work:backend:
  CLOSED: [2026-06-23 Tue 16:45]
  :LOGBOOK:
  CLOCK: [2026-06-23 Tue 14:00]--[2026-06-23 Tue 16:45] =>  2:45
  CLOCK: [2026-06-23 Tue 09:30]--[2026-06-23 Tue 11:00] =>  1:30
  :END:
  :PROPERTIES:
  :Effort:   3h
  :END:
✦
Clock persistence across sessions Add (org-clock-persistence-insinuate) and (setq org-clock-persist t) to your config.el so Emacs remembers the running clock between sessions. If you close Emacs mid-task, it will ask whether to resume or resolve the open clock on next start.
10 · Refiling

Refiling Tasks

Refiling moves a heading from one place to another — for instance, moving a quickly captured item from your inbox into the appropriate project file. Doom maps this to a convenient two-key sequence under the local leader.

Refile Keybindings
KeybindingAction
SPCmrr Refile the current heading (opens a fuzzy search of all headings in agenda files) Navigate the picker with C-n/C-p or type to filter.
C-c C-w Vanilla refile command (same underlying action)
SPCmrR Refile subtree — moves the entire subtree including all children
ℹ
GTD Inbox workflow A popular pattern is to keep a dedicated inbox.org as your fast-capture destination, then once a day refile everything from the inbox into your proper project files. Set (setq +org-capture-todo-file "inbox.org") in your config to route SPCXt there.
11 · Workflow

The Daily Workflow

The following morning-to-evening routine distils Doom's org capabilities into a repeatable daily ritual. Each step uses only the default keybindings covered in this guide.

1

Morning Review — Open Today's Agenda

Press SPCoaa to open the day agenda. Scan overdue items and deadlines approaching. Use vd to switch to a pure day-view if you want to focus only on today.

2

Brain Dump — Capture New Tasks Immediately

As ideas and tasks arrive throughout the day, press SPCXt to capture them instantly. Don't organize yet — just get them out of your head and into todo.org. This is your inbox. Return to your previous context in seconds.

3

Process — Refile and Schedule Captured Items

Visit todo.org or the agenda and process newly captured items. For each: set a state (SPCmt), assign a deadline or schedule (SPCmds / SPCmdd), tag it (SPCmq), and refile it (SPCmrr).

4

Execute — Clock In and Work

Pick your highest-priority task from the agenda. Navigate to it with RET, then clock in with SPCmci. Change its state to STRT while working on it.

5

Finish — Clock Out and Mark Done

When the task is complete, clock out (SPCmco) and advance the state to DONE by pressing SPCmt (or directly in the agenda with t). Doom records the CLOSED timestamp automatically.

6

Evening Review — Journal Entry

Press SPCXj to open a journal capture. Reflect on what you accomplished, what was blocked, and what carries over. This creates a date-tree entry in journal.org you can review any time.
